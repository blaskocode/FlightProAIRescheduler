# Memory Bank

This directory contains the project's memory bank - comprehensive documentation that persists across sessions and helps maintain context about the Flight Schedule Pro AI Rescheduler project.

## Structure

### Core Files

1. **`projectbrief.md`** - Foundation document
   - Project overview and goals
   - Problem statement and solution
   - Success metrics
   - Project scope and constraints

2. **`productContext.md`** - Product understanding
   - Why the project exists
   - Problems it solves
   - How it should work
   - User experience goals
   - Value proposition

3. **`activeContext.md`** - Current work focus
   - Current status and phase
   - Recent changes
   - Active priorities
   - Current blockers
   - Next milestones

4. **`systemPatterns.md`** - Architecture and design
   - System architecture overview
   - Key architectural decisions
   - Design patterns in use
   - Component relationships
   - Data flow patterns

5. **`techContext.md`** - Technical details
   - Technology stack
   - Development setup
   - Technical constraints
   - Dependencies
   - Performance targets

6. **`progress.md`** - Development status
   - What works
   - What's left to build
   - Current status
   - Known issues
   - Testing status

## Usage

These files are read at the start of every session to understand:
- What the project is about
- What has been built
- What needs to be done next
- How the system is designed
- What technologies are used

## Updates

The memory bank should be updated when:
- Significant features are completed
- Architecture decisions are made
- New patterns are established
- Project status changes
- User explicitly requests "update memory bank"

## Hierarchy

```
projectbrief.md (foundation)
    ├── productContext.md
    ├── systemPatterns.md
    └── techContext.md
            └── activeContext.md
                    └── progress.md
```
